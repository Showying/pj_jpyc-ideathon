# Project Charter: JPYC Idea-thon Submission

## 1. 目的 (North Star)
JPYCアイデアソンへの提出を行い、成果を残すこと。
単なる提出にとどまらず、実現性の高いソリューションとして企画・デモ・GitHubリポジトリを整備する。

## 2. 成果物 (Deliverables)
### 必須
- **企画概要**: タイトル、コンセプト
- **ターゲット業界**: 具体的な適用領域
- **解決したいPain**: 課題の深掘り
- **ソリューション概要**: ステーブルコイン（JPYC）活用の具体的仕組み
- **実現アプローチ (Feasibility)**: 技術的・運用的実現可能性
- **アクションプラン**: 3〜6ヶ月の実装・導入計画

### 加点・任意要素
- **企画書URL**: 詳細プレゼン資料など
- **デモ動画URL**: プロトタイプの動作デモ
- **GitHubリポジトリ**: 実装コード
- **実績・技術力アピール**: チーム/個人のケイパビリティ証明

## 3. スコープ (Scope)
- ✅ **In-Scope**:
    - アイデアの具体化・ブラッシュアップ
    - 企画書の作成（Markdown/Slide）
    - 技術検証（PoC）およびデモ作成
    - 提出用フォーム項目の埋め込み

- ❌ **Out-of-Scope** (現時点):
    - 本番環境への完全デプロイ（デモレベルでOK）
    - 法的コンプライアンスの完全クリア（あくまでアイデアソンとしてのFeasibility検討まで）

## 4. 成功条件 (Success Criteria)
- 期限内にすべての必須項目を埋めて提出完了すること
- デモまたはGitHubリポジトリを含め、加点要素を少なくとも1つ満たすこと

## 5. 主要リスク (Risks)
- **時間不足**: 企画と実装のバランス。PMOがバックログ管理で制御する。
- **技術的実現性**: JPYC固有の仕様（v2など）への適合。早期の調査が必要。
